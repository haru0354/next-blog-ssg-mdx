この記事では「記事・カテゴリ・メニューの作成手順」や、「ファイルの記述方法」についての紹介をしています。

もともとこのブログは私自身が使用する為に作成したものですが、使いたい人がいれば使えるように簡単ながら解説をしていきます。

基本的に難しいことはなく、<CustomText color="red">どの項目に何を記述をするのかさえ分かれば、簡単にブログが作成できる</CustomText>ようになっています。

## 目次

## カテゴリの作成

現状、最大で3階層まで対応をしており、親カテゴリと子カテゴリの2種類が作成可能です。

- 親カテゴリの作成手順
- 子カテゴリの作成手順
- カテゴリの記述方法

これらについて解説をしていきます。

### 親カテゴリの作成手順

1. mdFile ディレクトリの「categorySample」をコピー
2. 「mdFile > category」ディレクトリとへコピーしたサンプルファイルを入れる。
3. ファイル名を変更する(slugになる)。
4. 各項目を変更をする。
5. 「mdFile > article」ディレクトリにカテゴリのファイル名でフォルダを作成する

articleディレクトに作成したフォルダ内に個別記事を入れることで、その記事はカテゴリに属することになります。

### 子カテゴリの作成手順

1. mdFile ディレクトリの「categorySample」をコピー
2. mdFile > category ディレクトリに親カテゴリの.mdxを省いた名称でフォルダを作成
3. 作成したディレクトリにコピーしたサンプルファイルを入れる。
3. ファイル名を変更する(slugになる)。
4. 各項目を変更をする。
5. 「mdFile > article」ディレクトリの親カテゴリフォルダ内に.mdxを省いた子カテゴリの名称でフォルダを作成

articleディレクトに作成したフォルダ内に個別記事を入れることで、その記事はカテゴリに属することになります。

### 2種類のカテゴリページが作成可能

カテゴリは2種類のタイプのページが作成できるようになっています。

1. カテゴリ内へのカードリンクの表示ページ
2. カテゴリを記事にする(カードリンクも含む)

通常は1番が適用され、コンテンツを含めることで2番目が適用されます。

ようするに、カテゴリ内にもコンテンツや画像を含めることで、作りこんだカテゴリページを作成することができます。

サイト構造・SEO・ユーザービリティーを考慮すれば、カテゴリは大事なページとなります。

そのため、カテゴリページでもしっかりとキーワードを狙い、コンテンツを作りこめるようになっています。

### カテゴリの記述方法

カテゴリページで使用するファイルの記述についての解説です。

ファイル上部に各設定(Frontmatter)を記述する項目が用意されています。

- categoryName: カテゴリ名(サイドメニュー / TOP / パンくずリスト)
- title:  カテゴリページのタイトル(meta title / カテゴリページのH1タグ)
- description: カテゴリのmeta description
- date: 作成日付
- eyeCatchName:アイチャッチ画像のファイル名
- eyeCatchAlt: アイチャッチ画像のALT 

全項目で紹介をしたように、ファイル名はslug(URL)となっていますので、任意のファイル名の変更をしてください。

そして、設定エリアの下にコンテンツを記述することによって、その内容がページに反映されます。


## 記事の作成

通常のブログと同じように、各記事とどのカテゴリにも属さない固定記事の2種類が作成可能となっています。

- 記事の作成手順
- 記事の作成手順の例
- 固定記事(どのカテゴリにも属さない)の作成方法
- 記事の記述方法

これらについて解説をしていきます。

### 記事の作成手順

記事は以下の手順にて作成をします。

1. mdFile ディレクトリの「articleSample」をコピー
2. mdFile > article ディレクトリにフォルダを作成（フォルダ名は.mdxを除いた記事のカテゴリのファイル名）
3.コピーしたサンプルファイルを入れる。
3. ファイル名を変更する(slug(URL)になる)。
4. 各項目を変更をする。

作成するフォルダは、記事のカテゴリのファイル名から「.mdx」を除いた名前に必ずしてください。

#### 記事の作成手順の例

下記の例では「美白」カテゴリを作成して、その中に「美白化粧品」の記事を入れています。

1. 美白カテゴリを作成するため「bihaku.mdx」でカテゴリのファイルを作成
2. articleディレクトリに「bihaku」でフォルダを作成する。(手順1のファイル名と一致)
3. 「bihaku」フォルダに「articleSample」をコピー。
4. articleSampleのファイル名を記事のURL「cosmetics」に変更する。
5. ファイル内に美白化粧品のコンテンツを記載する。

上記の例では美白カテゴリ（URL:/bihaku）に美白化粧品の記事（URL:/bihaku/cosmetics）が入ります。
なお拡張子「.mdx」はURLに含まれません。

### 固定記事(どのカテゴリにも属さない)の作成方法

どのカテゴリにも属さない固定記事に関しては、「mdFile > article」ディレクトリ内に「articleSample」ファイルを入れてください。

この固定記事は「TOPページ ＞　固定ページ」となります。

まだ、どのカテゴリにも属さず、新着記事の項目などにも表示がされないようになっています。

主に、「WEBサイトの運営全般の内容」や「CTA目的とした記事」や「どのカテゴリにも属さない記事」などを想定しています。

### 記事の記述方法

記事ページで使用するファイルの記述についての解説です。

ファイル上部に各設定(Frontmatter)を記述する項目が用意されています。

- title: 記事タイトル(meta titleにもなります)
- date: 作成日付
- description: 記事のmeta description
- eyeCatchName: 00(拡張子なしのファイル名のみ記載)
- eyeCatchAlt: アイキャッチ画像のAlt

全項目で紹介をしたように、ファイル名はslug(URL)となっていますので、任意のファイル名の変更をしてください。

そして、設定エリアの下にコンテンツを記述することによって、その内容がページに反映されます。

## 目次の作成方法

目次に関しては自動で作成されるようになっています。

作成方法は簡単で「目次と記載された見出し」を記述すると、その見出しの記事の内容として目次が自動で挿入されます。

例)「## 目次」

上記の記述をすることで、テキストで目次と記載されたh2タグとその下に自動された目次が挿入されます。

但し、今後この方法は変更をする可能性があります。

わざわざ「目次の見出しを作るのが手間」、「見出しを増やしたくない」の以上の2点により、今後目次の追加方法については変更をする可能性があります。

## 追加できるメニュー

「mdFile > menu」内にある各ファイルを記載をすることで、様々なメニューを追加できるようになっています。

- グローバルメニュー(globalMenu.md)
- サイドバーのおすすめの記事(recommendArticle.md)
- サイドバーのリンク(参考文献・参考サイト)(link.md)

各ファイルの１つ目を記載すると、各メニューが表示されるようになります。

Word Pressのように誰でも簡単にブログが使えて、任意の項目を設定できるようにしたいという思いから、このような方法で各メニューが設置できるようにしています。

## デザインについて

各記事ページやカテゴリは装飾も行えるようになっています。

装飾の方法には下記の3種類があります。

- マークダウン形式
- HTML
- カスタムコンポーネント

カスタムコンポーネントを使用することで、簡単にデザインをすることができます。

但し、htmlなどの形式は注意する点もあります。

それぞれ、どのような方法で記述をするか紹介をします。

### マークダウン形式

記事の装飾はマークダウンでの形式での装飾が行えるようになっています。

- 「#」h1タグ、「##」h2タグ
- 「- 」リストタグ

などなど、マークダウンの形式での装飾が可能です。

マークダウンでの形式での装飾は様々なのがありますが、ここでは解説記事ではないので割愛させていただきます。

基本的には後程解説している、カスタムコンポーネントを利用すれば、マークダウンに知識がない人でも装飾できるので利用ください。

### HTML

htmlでの記述をすることも可能となっています。

但し、注意点としては<CustomText color="red">「HTMLタグに改行などを加えない」</CustomText>ことが必要となります。


```
// HTMLの記述方法

<p>ああ</p>

// 利用できない記述方法

<p>
    ああ
</p>
```

上記の記述のように、改行などを入れずに、そのままHTMLタグで囲うようにしてください。

改行などを入れることで上手く反映がされません。

また、全てのタグにデザインを現状は施せていません。
(随時追加中です)

CSSに理解のある人であれば「globals.CSS」に記述することで、任意の装飾を追加することができます。

### カスタムコンポーネント

カスタムコンポーネントとはあらかじめ作成された、デザインパーツを簡単に利用できる方法となります。

どのように利用するかは、各カスタムコンポーネントに違いがあるのですが、基本的には使いたいパーツを選んで、コピー貼り付けなどの方法で誰でも利用ができます。

また、覚えてしまえばコピーなどをせずとも、利用をすることは当然のことながらできます。

どんなデザインパーツがあるかや使い方は、「<CustomLink href="/sample2/sample/">デザインのサンプル</CustomLink>」をご覧ください。

## まとめ

基本的には私自身が使用する為に作成をしていますが、「誰でも簡単にBLOGの作成ができる」ように改修をしていっています。

過去の私の実績としては<CustomText color="red">「単一キーワード」や「ミドルワード」での上位表示経験</CustomText>がありますが、すでに私自身が利用したいと思える基本的な機能はすでに実装がされていると思います。

基本的にSEOを考慮したサイト構成にもなっているかと思います。

ただ、個人的にはまだまだ完成とは程遠いと思っており、下記のような機能を追加をしていく予定です。

- 誰でも使用できるようにマニュアルの実装
- デザインコンポーネントの追加
- 各装飾(CSS)の追加
- 機能の追加
- デザインを簡単変更(設定の記載でデザインを一括変更)

など、これからも改良を加えていく予定です。
